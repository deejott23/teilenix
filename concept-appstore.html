01

Technischer Weg in den Store

Capacitor + Vercel API ⭐ Empfohlen
Bestehenden Next.js Code als WebView wrappen. Supabase & Vercel Backend bleibt unverändert. App kommuniziert per HTTPS mit der Vercel API.
  • 95% Code-Wiederverwendung
  • Schnell umsetzbar
  • Günstig
  • Backend unverändert
  • Kein 100% nativer Look
  • Apple prüft strenger
  • Braucht native Extras
Aufwand: 3–5 Wochen Code-Änderung: ~10%
React Native (Expo)
Frontend komplett neu in React Native schreiben. Supabase Backend bleibt identisch. Echter nativer Look & Feel.
  • Echter nativer Look
  • Beste Performance
  • Bessere Review-Chancen
  • 12–20 Wochen Aufwand
  • Frontend komplett neu
  • Sehr hohe Kosten
  • Doppelter Code-Stand
Aufwand: 12–20 Wochen
PWA (kein Store)
Progressive Web App — kein App Store, kein Apple-Abo. Nutzer speichern die Website auf dem Homescreen.
  • 0 Zusatzaufwand
  • Sofort verfügbar
  • Keine Provision
  • Kein App Store
  • Kein Push auf iOS
  • Eingeschränkte Features
  • Weniger Vertrauen bei Nutzern
Aufwand: 1–2 Wochen
⚠️
Wichtig für Capacitor: Apple lehnt reine WebView-Wrapper ab (Guideline 4.2). Mindestens 3 native Features müssen integriert werden: Push Notifications, Haptic Feedback und Share Sheet für Einladungslinks.

Häufige Ablehnungsgründe

🚫
Guideline 4.2 — Minimum Functionality: Reine WebView-Wrapper ohne native Mehrwerte werden konsequent abgelehnt.
→ Lösung: Push Notifications, Haptic Feedback und Share Sheet integrieren
🚫
Fehlende Datenschutzerklärung: Eine öffentlich erreichbare Privacy Policy URL ist Pflicht.
→ Lösung: teilenix.app/privacy hosten (einfache statische Seite)
🚫
In-App-Käufe außerhalb von Apple IAP: Wenn Monetarisierung geplant, müssen Käufe über Apple laufen.
→ Lösung: RevenueCat SDK für iOS In-App Purchases nutzen
⚠️
Screenshots aus dem Simulator: Apple verlangt echte Screenshots von einem echten iPhone.
→ Lösung: iPhone 15 Pro (oder neueres) für Store-Screenshots verwenden

App Store Review — wie schwer ist die Freigabe?

Schwierigkeit
MITTEL
Ersteinreichung: 1–7 Tage
Updates: 1–2 Tage
Einschätzung
TeileniX ist eine klar definierte Utility-App mit konkretem, nachvollziehbarem Nutzen — das erhöht die Freigabe-Wahrscheinlichkeit erheblich. Es gibt keine rechtlichen Graubereiche, keine Finanztransaktionen und keine nutzergenerierten Inhalte. Die größte reale Hürde ist die WebView-Problematik.
WebView ohne native FeaturesApple lehnt Apps ab, die nur eine Website im Vollbild anzeigen. Ohne mindestens 3 spürbare native Features (Push, Haptics, Share) ist Ablehnung sehr wahrscheinlich.
Fehlende IAP-Integration bei MonetarisierungFalls Abo geplant ist und es nicht über Apple IAP läuft, folgt sofortige Ablehnung.
Klare Funktion — kein Gray-Area ContentReisekostenabrechnung ist eine eindeutig nützliche Kategorie ohne rechtliche Risiken.
Keine sensiblen FinanzdatenTeileniX führt keine Transaktionen durch, sondern nur Aufzeichnungen — geringere Prüfintensität.
Kein User Generated Content (UGC)Kein öffentliches Forum, keine Kommentarfunktion — weniger Compliance-Aufwand.
✅
Strategie für erfolgreiche Ersteinreichung: 3 native Features einbauen (Push, Haptics, Share Sheet) · Privacy Policy unter eigenem URL hosten · Support-Kontakt in der App verlinken · Echte iPhone-Screenshots erstellen · Klare App-Beschreibung: "Reisekostenabrechnung für Gruppen"

06

Technische Umsetzung Abo-Modell

⚠️
Apple-Regel: Nutzer dürfen für In-App-Features nicht auf externe Zahlungsseiten (PayPal, Stripe direkt) weitergeleitet werden. Alle Käufe für App-Features MÜSSEN über Apple IAP laufen → 15–30% Provision ist unvermeidbar.
Ausnahme: Die Web-Version kann weiter direkt über Stripe abrechnen — kein Apple-Cut für Web-Abos.
RevenueCat Integration — Schritt für Schritt
  1. RevenueCat Account anlegen Kostenlos bis 2.500 USD MRR. Verbindet App Store Connect mit der eigenen App und übernimmt das komplette Subscription-Management.
  2. In-App-Purchase in App Store Connect anlegen Zwei Produkte: teilenix_premium_monthly und teilenix_premium_yearly. Preisgestaltung über Apples Preisstufen.
  3. RevenueCat SDK in Capacitor einbinden Das SDK übernimmt Kauf-Flow, Restore, Receipt-Validierung und Webhook-Benachrichtigungen automatisch.
  4. Supabase-Schema erweitern profiles Tabelle: is_premium boolean + premium_expires_at timestamptz hinzufügen. Migration: ~5 Minuten.
  5. RevenueCat Webhook → eigener API-Endpoint Bei erfolgreicher Zahlung: RevenueCat feuert Webhook → POST /api/webhooks/revenuecat → Supabase is_premium auf true setzen.
  6. Feature-Gates im Frontend implementieren Vor dem Erstellen einer 4. Reise: Premium-Check → Paywall-Screen anzeigen. Limit-Logik serverseitig validieren (nicht nur im Client).
  7. "Premium upgraden"-Screen in der App Zeigt Preise aus dem App Store (dynamisch via RevenueCat SDK), Kaufbutton, Restore-Button für bereits zahlende Nutzer.

Empfehlung & Roadmap

Phase 1 · Monat 1–2
Vorbereitung & Setup
  • Apple Developer Account anlegen (99 USD)
  • Mac beschaffen falls nötig (Mac Mini M2 ab ~700€)
  • Privacy Policy + Impressum als statische Seite hosten
  • App Icon 1024×1024px finalisieren (auch für App Store nötig)
  • Capacitor in das Next.js-Projekt integrieren
  • 3 native Features: Push Notifications, Haptic Feedback, Share Sheet
Phase 2 · Monat 2–3
Build, Test & Beta
  • iOS Build via Xcode erstellen
  • TestFlight Beta — bis zu 10.000 Tester kostenlos einladen
  • Bugfixing auf echten Geräten (iPhone 14/15 zum Testen)
  • Screenshots auf echtem iPhone 15 Pro für Store-Listing
  • App Store Connect: Listing, Beschreibung, Keywords, Alterseinstufung
Phase 3 · Monat 3
Store-Einreichung & Launch
  • Finale Einreichung in App Store Connect
  • Review-Wartezeit: 1–7 Tage (Durchschnitt ~2 Tage)
  • Bei Ablehnung: Anpassung nach Review-Feedback, erneute Einreichung
  • Go-Live: App erscheint im deutschen App Store
Phase 4 · Monat 4+
Monetarisierung aktivieren
  • RevenueCat SDK integrieren + Webhook-Endpoint bauen
  • Freemium-Limits im Backend implementieren (serverseitig!)
  • Premium-Abo-Produkte in App Store Connect anlegen
  • Paywall-Screen in der App gestalten
  • Update einreichen → Review 1–2 Tage → Monetarisierung live
  • Small Business Program bei Apple beantragen (15% statt 30%)

Fazit & Empfehlung

TeileniX hat das Potenzial für einen erfolgreichen App Store Launch. Die größte Hürde ist nicht Apples Review-Prozess an sich — sondern die technische Notwendigkeit, mindestens 3 spürbare native Features in die App zu integrieren, um nicht als reiner WebView-Wrapper abgelehnt zu werden.

Empfehlung: Capacitor-Ansatz + Freemium-Abo-Modell. Dieser Weg maximiert die Code-Wiederverwendung, minimiert den Aufwand bis zum Launch und schafft die Basis für nachhaltigen, wiederkehrenden Umsatz. Die Web-Version bleibt weiterhin vollständig funktional und kann parallel ohne Apple-Provision über Stripe monetarisiert werden.
