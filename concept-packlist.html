Konzeptvergleich

Kriterium
🟢 Konzept A — Drei Bereiche
🟡 Konzept B — Smart-Feed
Mentales Modell
✓ Sehr klar — drei getrennte "Räume"
~ Braucht kurze Eingewöhnung für Farbkodierung
Gesamtüberblick
− Gruppe-sucht-Items brauchen Scrollen
✓ Alle Item-Typen auf einen Blick sichtbar
Neue User
✓ Intuitiv ohne Erklärung
− Farbstreifen muss man einmal lernen
Geschwindigkeit
~ Section-Toggle nötig für versteckte Typen
✓ Filter-Chips = sofortiger Typ-Wechsel
Claim-Interaktion
✓ Gut sichtbar, klarer Call-to-Action
✓ Kompakt integriert, kein Scrollen nötig
Implementierungsaufwand
✓ Einfacher — klare Struktur, weniger Edge Cases
− Filter-Logik + Farbsystem etwas komplexer
Skalierbarkeit (>20 Items)
− Lange Sections, viel Scrollen
✓ Filter-Chips reduzieren sichtbare Liste
Ästhetik / Konsistenz
✓ Farbige Sections passen gut zum TeileniX-Stil
✓ Moderner, dichter — ähnlich Notion/Todoist
Empfehlung: Konzept A für den Start — schneller umsetzbar, sofort verständlich, passt perfekt zum bestehenden TeileniX-Designprinzip der klaren Trennung (ähnlich wie Ausgaben-Kategorien). Konzept B als Version 2 wenn die Listen länger werden und Nutzer bereits vertraut sind.
